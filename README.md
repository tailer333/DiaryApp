# DiaryApp — ежедневник (Android)

Мобильное приложение-ежедневник на Kotlin: календарь, почасовая сетка дня, детали и создание дел. Учебный / тестовый проект.

## Возможности

- **Расписание на день** — выбор даты (Material DatePicker), таблица из 24 часовых интервалов (`00:00–01:00` … `23:00–24:00`). В слоте отображаются дела с **названием и временем**; нажатие открывает подробности.
- **Экран дела** — название, дата и время (начало — конец), текстовое описание.
- **Создание дела** — название, дата, время начала и окончания, краткое описание; сохранение в локальную БД.

## Стек

- **Kotlin**, **Jetpack Compose**, Material 3  
- **Room** — локальное хранение  
- **Navigation Compose**  
- **MVVM** + репозиторий  
- **Сервисный слой** — `TaskScheduleService` (построение почасовой сетки и пересечения с интервалами дел)  
- **Gson** — модель JSON для списка дел (см. ниже)  
- **Unit-тесты** — сервис расписания, разбор timestamp в JSON

**minSdk 26** (Android 8.0+), ориентация — портретная.

## Формат JSON (модель дела)

Пример структуры записи:

```json
{
  "id": 1,
  "date_start": "147600000",
  "date_finish": "147610000",
  "name": "My task",
  "description": "Описание моего дела"
}
```

Поля `date_start` и `date_finish` — timestamp в виде строки: значения **меньше 10¹²** трактуются как **секунды** Unix, иначе — **миллисекунды**. Классы: `TaskJsonDto`, `TaskJsonCodec`.

## Как запустить

1. Установите [Android Studio](https://developer.android.com/studio) (желательно последняя стабильная версия).  
2. **File → Open** — выберите корень этого репозитория (папка с `settings.gradle.kts`).  
3. Дождитесь синхронизации Gradle.  
4. Создайте эмулятор (**Device Manager**) или подключите устройство с отладкой по USB.  
5. Нажмите **Run** (▶) для конфигурации `app`.

При первом запуске в базу добавляются примеры дел на текущий день.

## Сборка из пути с не-ASCII (Windows)

Если проект лежит в папке с кириллицей и т.п., в `gradle.properties` включено:

`android.overridePathCheck=true`

Для надёжной работы инструментов можно дополнительно перенести проект в путь только с латиницей (например `C:\dev\DiaryApp`).
